Prompt for Friday, Jan. 30. Scalable Vector Graphics

SVG is an XML specification of vector graphics. Visual elements are represented as tags, which have styles and attributes.
1. Polylines. How does SVG understand coordinates?
2. Paths. This is the most complicated SVG element, but also one of the most useful. It contains a tiny programming language.
3. Transformations are another useful (but a bit tricky) tool. How do we stretch, rotate, and shift elements, and how does the order of these operations affect the result?

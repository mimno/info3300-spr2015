Prompt for Friday, Jan. 24. What's different about Javascript?

At the end of each class I'm going to post a "prompt" for the next class. This one will be highlighted at the end of our first class, and will form the basis for our discussion on Friday. As we build up complexity during the semester, these will start to look more like templates for interesting mini-projects.
This class will work almost entirely in Javascript. There are several idioms in Javascript that we'll use a lot, but that you may not be familiar with from other languages. Don't use any additional libraries, we want to do things the hard way first!
Javascript has a simple type system. Use the typeof function to find the type of the following:
  • 3
  • 3.14159
  • "3"
  • true
  • "true"
Javascript is object-oriented, but in a simpler way than you might be used to. An object is an unordered set of key/value pairs. Write an object literal with variables "x" and "y", and values of your choice. Can keys be of any type?
Javascript is functional, which means that functions are values. There are two different ways of defining functions.
Some functions take other functions as inputs. Other functions return functions.

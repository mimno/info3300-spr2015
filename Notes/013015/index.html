Prompt for Friday, Jan. 30. Scalable Vector Graphics

SVG is an XML specification of vector graphics. Visual elements are represented as tags, which have styles and attributes.
We looked at this visualization of NYC CitiBike ridership data. Look at how the author starts by displaying just seven data points, and ends with around 20,000 data points.
1. Polylines. SVG interprets the points attribute as a string containing at least one pair of x,y coordinates. Coordinates can be integers or floating point numbers. Commas between coordinates are optional, but helpful for (human) readers. The order of points determines where the line goes and what the shape of the fill looks like. Try reversing the middle two coordinates.
2. Paths. This is the most complicated SVG element, but also one of the most useful. It contains a tiny programming language. Each command is a single letter followed by (usually) an x,y coordinate. Capital letters represent absolute coordinates; lowercase letters represent offsets from the current position. M moves the "pen" to a location. L draws a line from the current location. Z draws a line to the starting point, so we can close a shape without remembering where we started. See the full specification for other useful commands, like horizontal and vertical lines, arcs (useful for pie charts), and curves.
3. Transformations are a useful (but a bit tricky) tool. Here's a reference. Any SVG element can have a transform attribute. To apply the same transform on multiple tags nest them inside a <g> tag. Use scale(factor) to stretch a shape. Use translate(x,y) to move a shape. Use rotate(degrees) to rotate a shape. Your transform attribute can contain more than one of these. The order of operations matters. Guessing what will happen can be difficult. Try several combinations and see what you get.
